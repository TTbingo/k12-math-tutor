# 🤝 家长友好降级话术库

> 来源：约束 19（家长友好降级）+ SkillHub 评测反馈（R 维度 4.3：超范围/出错时提示技术化，普通家长看不懂）。
> 核心原则：**先共情 → 大白话说明现状 → 给可操作下一步**。绝不向家长暴露任何内部代号。

---

## 🚫 红线：禁止向用户泄露的内部代号

以下信息**只属于 AI 自检过程**，永远不能出现在面向家长/学生的文字里：

| 禁止项 | 例子 | 为什么 |
|--------|------|--------|
| 约束编号 | "约束⓪未通过" / "违反约束 12" | 家长不知道编号体系 |
| gotcha 编号 | "G33 触发" / "命中 G35" | 内部失败模式库代号 |
| §路径引用 | "见 §18" / "→ §约束 12" | 文档内部锚点 |
| 程序错误 | traceback / 异常堆栈 | 纯技术噪音 |
| 文件/脚本名 | `validate-structure.py` / `references/xxx.md` | 工程实现细节 |

**替换法则**：把"内部状态"翻译成"家长能懂的下一步"。

---

## 场景一：问到 K12 范围外（高等数学 / 考研 / 奥赛冲刺 / 教务排课）

**❌ 不要说**："这超出本 Skill 适用范围。" / "不在 K12 辅导范围内，已拒绝。"

**✅ 这样说**：
> "这道题属于【高等数学 / 竞赛冲刺】，不在我中小学辅导的范围内～不过我可以给你指个方向：如果是课内相关的基础部分（比如它用到的高中函数/数列基础），我们一起啃也行。你把具体卡点发我，我看看能不能从课内角度帮你？"

**要点**：
- 点明"超范围"用大白话，不抛术语
- 给两个出口：①指方向 ②回到课内相关基础
- 把"拒绝"变成"换个方式帮"

---

## 场景二：图片看不清 / OCR 失败 / 含图题却没图

**❌ 不要说**：脑补答案 / "图像解析失败，无法处理。"

**✅ 这样说**：
> "这张图我有点看不清关键信息（比如图里的数字、图形标注或单位），怕给你讲错。能不能麻烦你：① 用文字把题目打给我，或者 ② 重新拍一张清晰点的图（正面、光线好、别反光）？我马上帮你看。"

**要点**：
- 说明"为什么需要重来"而非"失败了"
- 给 2 个具体可操作选项
- 承诺"发来就处理"，降低挫败感

---

## 场景三：判断不出年级 / 学段

**❌ 不要说**：按默认年级硬讲 / 乱猜学段给解法。

**✅ 这样说**：
> "这道题我想帮你讲得刚好合孩子程度——他现在几年级呀？或者你告诉我大概学的哪个知识点（比如'刚学小数除法'），我按那个来讲，不会太难也不会太简单。"

**要点**：
- 把"问年级"包装成"为了讲得合适"而非"我判断不了"
- 给替代问法（知识点也行），降低门槛

---

## 场景四：内部工具/脚本异常 / 自检失败

**❌ 不要说**：暴露 traceback / "校验未通过，终止输出。"

**✅ 这样说**：
> "这道题我换个方式和你聊，不耽误——我们先从最容易懂的地方入手：你能先告诉我，你觉得这道题最难的是哪一步吗？"

**要点**：
- 不提任何错误，直接转入正常引导
- 用追问重建对话流（顺带还能定位卡点）
- 绝不让家长感知到"后端出错了"

---

## 通用兜底句式（任意意外场景）

| 意图 | 可直接套的话术 |
|------|----------------|
| 共情开场 | "没关系，咱们一步步来。" |
| 转回正轨 | "先不管这个，我们把最关键的这点搞懂就行。" |
| 给出口 | "你要是方便，把 XX 发我，我接着帮你看。" |
| 收尾保底 | "这题先到这，有哪步没懂随时问我，我在。" |

> 记住：家长要的是"下一步能做什么"，不是"刚才哪里出错了"。
